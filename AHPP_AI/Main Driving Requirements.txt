Product Requirements Document: AI Racing Driver System for Live for Speed
Overview
The AI Racing Driver System provides autonomous driving for LFS vehicles using InSim control packets, recorded route data, and real‑time telemetry. It supports multi‑route traffic networks (main, alternate, pit entry, detours), configurable lane changes, recovery behaviors, and in‑game visualization/debug tooling. The system also manages AI population counts and route allocations based on live server occupancy.

Core Features
1. Route Capture, Management, and Validation
Recorded Routes (JSON): Capture player‑driven routes with per‑node metadata (position, speed, heading, throttle, brake, steering).

Route Metadata: Track type (main loop, alternate main, pit entry, detour), attach/rejoin indices, default speed limits, AI weights/targets, and enablement.

Route Library: Load, and normalize, the standard routes are (Main Route, Main Alt Route, Pit to Main). Only Main Route and Pit to Main are required for AI to drive on it.

Route Validation: Validate recorded routes on load and surface issues via logging + chat summaries.

2. Multi‑Route Navigation
Main + Alternate Lanes: Support a primary route and optional alternate main lane that is two main loops, that can be merged between the two lanes if aligned close enough.

Detour / Branch Routes: Dynamically discover detours from recorded routes and integrate into the path manager.

Spawn/Pit Entry Routes: Dedicated spawn route used for pit entry or staging that can be joined by closest point from spawning car to the main route.

3. Dynamic Lane Change System
Probabilistic Merges: Configurable merge chance, cooldowns, and check intervals for moving between main/alternate lanes.

Safety Checks: Parallel distance, heading alignment, and speed thresholds guard merges.

Signaling: Turn‑signal lead time and minimum duration before committing lane transitions and turns off when the merge is complete.

Transition Paths: Generate a temporary interpolation path for smooth lane changes.

4. Vehicle Control + Physics‑Aware Driving
Pure Pursuit Steering: Optional pure‑pursuit controller with speed‑scaled lookahead and steering damping.

Gearbox/Clutch Management: Human‑like shifts, clutch timing, and stall prevention.

Speed + Waypoint Control: Speed targeting per waypoint with dynamic proximity thresholds.

5. Collision Awareness & Recovery
Forward Collision Detection: Detect vehicles ahead within a corridor and apply stopping behavior.

Recovery State Machine: Multi‑stage reverse/steer/forward recovery with escalation to pit/spectate.

6. Telemetry, Debugging & Visualization
InSim UI Panels: Controls for AI spawning, recording, route selection, layout visualization, and InSim status.

Layout Visualization: Render waypoints/routes into LFS layout objects with sampling to stay under limits.

Active Waypoint Markers: Optional periodic marker placement during runtime.

Performance Tracking: Rolling performance snapshots for MCI/AII handling.

7. AI Population Management
Auto‑Populate: Scale AI count based on human occupancy, min/max caps, and fill ratio.

Route Distribution: Allocate AI across routes using metadata targets/weights.

Technical Requirements
InSim Integration
Control: Use IS_AIC for throttle/brake/steering/gear/clutch and ignition.

Telemetry: Process IS_MCI and IS_AII for live car state and AI info.

Layout Objects: Use IS_AXM for waypoint/route visualization and markers.

System Architecture
Modular components: AIController, WaypointManager, RouteLibrary, PathManager, AIDriver, AIPopulationManager, UI/Debug UI.

Thread‑safe reconciliation for AI population adjustments.

Sub‑100ms control loop targets with configurable log windows.

Configuration Requirements
Population: max players, reserved slots, AI fill ratio, batch sizes.

Routes: main/alt/spawn route names, detour discovery.

Lane Change: merge chance, cooldowns, safety bounds, signaling timing.

Steering: damping, deadzone, pure pursuit tuning.

Recovery: stall detection thresholds and back‑off parameters.

Visualization: sampling detail, active markers, auto visualize.

Functional Requirements
Route Recording + Editing
Record routes with speed/heading snapshots and optional speed limits.

Save to JSON under route library; auto‑generate templates when missing.

Support attach/rejoin indices to define branch mapping.

Navigation
Follow recorded routes with speed‑aware waypoint thresholds.

Support alternate main lane and detour branch selection.

Lane Change
Evaluate merge feasibility based on geometry, speed, and timing windows.

Signal before lane change and transition through generated path.

Return to main route when branch requirements end.

AI Control + Safety
Implement pure pursuit and heading‑error steering fallback.

Detect potential collisions and stop as necessary.

Recover from stalls and stuck conditions with multi‑stage behavior.

UI & Debugging
In‑game UI controls for AI spawning, auto‑population toggle, route recording, layout controls.

Visualize active route selections and layout nodes with sampling controls.

Performance logging for MCI/AII processing.

Performance Requirements
Support multiple AI concurrently with auto‑population scaling.

Maintain waypoint tracking within configurable threshold multipliers.

Recovery actions complete within configured windows.

Telemetry processing at or above 10Hz.

Constraints & Limitations
Must operate within LFS InSim packet limits and layout object caps.

Visualization constrained by LFS layout object limits (sampling required).

AI behavior bounded by LFS physics and network latency.
